Sizzle是一个非常困难的靶机，知识点涉及smb匿名访问、SQL server获取凭证、SQL server命令执行、NTLM哈希破解、信息收集等。smb匿名访问共享名后将其中excel下载下来，其中包含SQL server数据库的密码，通过该密码登陆数据库，执行xp_dirtree函数拿到用户访问的NTLM哈希，破解后成功使用该账号密码再次登陆数据库，使用该用户开启xp_cmdshell函数从而完成命令执行成功拿到用户权限，在靶机中找到组策略文件当中的管理员密码，破解后成功获取管理员权限。感兴趣的同学可以在HackTheBox中进行学习。

0x01 侦查

端口探测

首先通过nmap对目标进行端口扫描



扫描结果显示目标开放了135、445、1433、5985等端口

445端口

使用 smbmap 对 smb 服务进行初步扫描


使用 smbclient 查看共享名称，在其中发现Reports


使用 smbclient 连接共享名Reports，发现其中存在一个 excel 文件


下载该文件并移到本地环境使用 excel 软件打开，但是文件为空

使用 strings 查看该文件也没什么发现


继续使用 binwalk 将该文件中的各种源文件提取出来


通过查找后在vbaProject.bin中发现端倪


其中存在 SQLServer 的账号密码以及连接信息

0x02 上线[mssql-svc]

SQL Server登陆

既然有了数据库连接信息，那么可使用 mssqlclient.py 连接目标的 MSSQL 数据库


登陆成功后对数据库进行信息收集，包括库名和权限



查询数据库中的表名


经过查询后发现目标数据库并没有什么可利用的数据。

smb获取ntlm哈希

而在之前做 Giddy 靶机的时候遇到过xp_dirtree命令，尝试使用它来利用 smb 进行身份验证，在本地开启监听从而获取 ntlm 哈希值。首先在本地开启 responder

通过 SQL 语句以服务用户的身份连接本地的 smb 共享


在 responder 中成功获取到 ntlm 哈希值

ntlm哈希破解

使用 hashcat 破解 ntlm 哈希值


成功拿到密码为corporate568，那么我们就可以使用新的身份再次登录数据库


查询用户 mssql-svc 的所有权限，发现该用户相比之前的用户拥有更多有用的权限
