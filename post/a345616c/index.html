0x02 上线[www-data]

文件上传绕过

综上我们可以发现两个文件上传点，分别为 torrent 以及 torrent 对应图片。显然我们对图片上传点更感兴趣，但直接上传 webshell 目标会返回invalid file，尝试使用一些常见的绕过方法进行测试

  • 修改文件扩展名
  • 修改 Content-Type
  • 修改文件上传内容

上传图片使用 BurpSuite 截取数据包并发送至 Repeater 进行测试

尝试直接修改文件扩展名为.php、Content-Type为application/x-php，点击发送后无法上传

尝试修改Content-Type值为image/png后上传成功，猜测 web 应用程序主要判断Content-type类型，将其修改为正常的图片类型可绕过上传限制

为了验证这一猜想，我们只修改图片上传数据包的Content-type的值。结果同样也无法上传，说明猜想正确

0x03 权限提升[root]

信息收集

枚举当前用户目录，发现文件motd.legal-displayed，虽然该文件是空的，但是这类文件可能会导致远程代码执行漏洞的产生

参考文章：https://www.exploit-db.com/exploits/14339


MODT提权

通过阅读该漏洞的利用代码，该漏洞出现于用户登录时会调用 PAM 对~/.cache设置目录权限。由于当前环境为反弹shell，因此我们无法触发该漏洞，但我们可以写入 SSH 密钥用于登录，将~/.cache替换为需要的文件符号链接，比如passwd。替换完成后再次登录该文件会被当前用户调用，从而完成权限提升。

SSH登录

查看当前用户对~/.cache目录的权限，该目录中的motd.legal-displayed归 george 所有，权限为可读可写，因此我们无法删除该文件

在当前用户的主目录下创建.ssh目录并使用 ssh-keygen 生成 SSH 密钥对

将公钥复制到authorized_keys并设置权限为600

与此同时当前目录中并没有.cache目录

复制 www-data 用户私钥并连接目标的ssh

在主目录下出现~/.cache目录并在其中发现motd.legal-displayed文件

添加用户

删除~/.cache目录并将其替换为/etc/passwd的文件符号链接

再次登录ssh，发现/etc/passwd权限已经归 www-data 所有

既然有了写入权限，我们可以尝试添加 root 用户，在此之前我们还需要获取密码的哈希值

添加用户至/etc/passwd，其中用户和密码都是 mac，用户和用户组都为0代表 root，描述为 pwned，主目录是/root，shell 是/bin/bash

使用su命令获取 root 权限

在/root下寻找第二个flag，成功找到第二个flag

脚本分析

脚本分别定义了backup()、restore()、key_create()、key_remove()、own()、bye()等一堆函数，执行流程如下：

  • key_create()创建SSH密钥对以便当前用户通过私钥无密码登录
  • backup()备份~/.cache目录
  • own()会删除~/.cache目录并创建文件符号链接，通过该函数分别指向/etc/passwd、/etc/shadow，然后断开连接验证当前用户是否已拥有目标文件的写入权限
  • restore()用来恢复~/.cache目录
  • key_remove()则会清理刚刚添加的SSH密钥
  • 清理痕迹完成后再自动启动 root 权限的 shell

0x04 总结

通过对站点进行目录扫描后我们发现三个目录，分别是phpinfo界面、重命名API以及提供 Torrent Hoster 实例的应用程序。在应用中注册用户后发现其中存在种子上传点，与此同时还能上传图片。在尝试对图片上传点上传木马后发现应用存在防御机制，修改Content-Type可以绕过限制。使用木马执行命令成功收到反弹shell。

上线后当前权限为www-data，通过信息收集发现MOTD文件，而该文件能够帮助我们完成权限提升。该漏洞主要是因为用户登录时会调用 PAM 对~/.cache设置目录权限，因此需要我们生成密钥对来登录目标用户ssh，将~/.cache替换为管理员执行命令/etc/passwd，再次登录后会调用PAM重设该命令所有者，从而我们就能够添加管理员用户完成权限提升。当然还有个经典方法：脏牛提权，这个方式就不再赘述了，有想要了解的朋友可以直接看操作步骤。
