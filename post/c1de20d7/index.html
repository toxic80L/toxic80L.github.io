0x02 上线[tolis]

ColdFusion漏洞检测

使用 searchsploit 查找 ColdFusion 8 的相关漏洞，其中有利用价值的就是文件上传漏洞16788.rb和目录遍历漏洞16641.py

FCK编辑器文件上传

EXP分析

首先分析 FCK 编辑器文件上传漏洞的EXP16788.rb，它共发出两次 http 请求，第一次是以 POST 方式向 FCK 目录上传文件，第二次使用 GET 方式请求/userfiles/file触发 payload 完成上线。

上传木马

根据漏洞利用代码，我们首先使用 msf 生成木马

通过 curl 以 POST 方式上传木马

请求完成后访问/userfiles目录，但是其中并没有上传的木马

绕过限制获取反弹shell

这主要是由于网站进行了过滤，需要修改上传文件的后缀名和 Content-Type，同时将目录设置为木马名称，通过00截断完成木马上传

访问木马成功获取反弹shell

0x03 权限提升[system]

0x04 总结

通过信息收集已知目标靶机中的8500端口开放应用 ColdFusion 8，该应用存在许多历史漏洞，通过 searchsploit 搜索我们可发现两种渗透路径：一是利用 ColdFusion 自带的FCK编辑器存在的未授权文件上传漏洞直接上传木马；二是利用 ColdFusion 的目录穿越拿到管理员密码，登录后台通过文件接口上传木马。

以上两种方式均可获取用户权限，第一种方法虽然简单，但是站点存在限制，需要配合00截断才能完成文件上传，第二种方式需要破解密码哈希值，一旦哈希值过于复杂也无法拿到明文密码登录后台。在靶机中存在多个内核漏洞，可通过 ms10-059 等漏洞拿到系统权限。
