HTB: Tally

Tally是一个困难的靶机，知识点涉及SharePoint信息收集、KeePass解密、zip密码破解、SQL server反弹shell、烂土豆内核提权等。SharePoint、ftp以及smb的信息收集获取到数据库密码，通过SQL server的命令执行可获取权限。感兴趣的同学可以在HackTheBox中进行学习。

0x01 侦查

端口探测

首先通过nmap对目标进行端口扫描




扫描结果显示目标开放许多端口，包括21、80、81、135、139、445、1433等

80端口

SharePoint是Microsoft Office集成的用于文档管理、存储以及共享的web协作平台

访问http://10.10.10.59，是一个 SharePoint 应用

对网站使用专门的 SharePoint 字典进行目录扫描


扫描结果显示两个重要的文件，分别是/sitedirectory/_layouts/viewlsts.aspx和/shared documents/forms/allitems.aspx

SharePoint信息收集

访问viewlsts.aspx和allitems.aspx界面如下


查看 ftp-details，成功获取主机名、工作组以及密码信息

访问10.10.10.59/SitePages/Forms/AllPages.aspx

查看 Finance Team，成功获取登陆账号ftp_user

KeePass密码破解

通过 ftp_user/UTDRSCH53c”$6hys 登录目标ftp


进入 User 目录后查看 Sarah 用户目录，发现 notes.txt 并下载


在本地查看 notes.txt，在其中提示已完成事项为安装 SharePoint，待办事项为卸载 SQL server 2016


之后进入 Tim 用户目录下，发现 tim.kdbx 并下载。其中 kdbx 文件就显得比较眼熟，在之前靶机 Jeeves 中也遇到过


使用 keepss2john 将其转换成可识别的哈希值


使用 john 进行哈希破解，但是一直没破解成功

原因是通过ftp下载文件时传输出错，靶机中的 tim.kdbx 有 2222 bytes，到了本地却只有 2220 bytes

我们可以通过 http 协议将 tim.kdbx 下载下来

通过两者对比后发现确实相差 2 bytes

重新使用 hashcat 进行破解


破解成功后密码为 simplementeyo，使用 keepass2 打开 tim.kdbx


按ctrl+h显示密码，获得smb共享路径为 ACCT，用户名为 Finance，密码为 Acc0unting

0x02 smb登陆

信息收集

使用获取到的账号密码通过 smbclient 登录目标


成功登录smb服务后开始寻找有用信息


查看 conn-info.txt

拿到旧的 SQL server 数据库连接信息

zip密码破解

经过搜索后在\zz_Migration\Backup\20170808\orcharddb\下发现 orcharddb.zip，下载后无法直接解压

使用 frackzip 对 zip 压缩文件进行密码破解


成功破解密码为 Acc0unting，与登陆密码一致。于是使用 unzip 解压压缩包

查看解压出来的sql文件发现账号密码为 admin/Finance2，但这是 Orchard CMS 的登陆信息，实际上它已经被替换为 SharePoint

获取SQL server凭证

在\zz_Migration\Binaries\New folder\中发现exe文件

使用 strings 进行查看


成功拿到 SQL server 的账号密码为 sa/GWE3V65#6KFH93@4GWTG2G，是数据库的最高权限账号

0x03 上线[sarah]

SQL server数据库命令执行

使用 spsh 通过以上账号密码连接 SQL server 数据库


开启命令执行 xp_cmdshell

通过 xp_cmdshell 查看本机信息



由于这个SQL窗口每次都需要重新配置，我们可以尝试使用 MSF 中的 mssql_exec 来拿到稳定的命令窗口

MSF利用

首先开启在本地监听


利用命令模块 mssql_exec 获取反弹shell



但是目标并没有反弹，考虑使用 nishang 中的 Invoke-PowerShellTcp.ps1 脚本进行反弹

将反弹的地址和端口添加到该脚本下，同时开启 http 服务

在本地监听5555端口

设置SQL语句完成命令执行


成功获取到反弹shell，在当前用户桌面寻找第一个flag


成功获取到第一个flag，与此同时发现提示信息：待办事项分别为安装更新和查看防火墙是否可用
