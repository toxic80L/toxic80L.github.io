HTB: Helpline

Helpline是一个困难的靶机，知识点涉及EFS解密和ME SDP的多个漏洞，包括XXE漏洞、LFI\任意文件下载漏洞、身份认证绕过漏洞以及远程代码执行漏洞等。其中虽然ME SDP相关漏洞可以上线系统最高权限，但是需要解密EFS才能读取关键flag。感兴趣的同学可以在HackTheBox中进行学习。

0x01 侦查

端口探测



通过 nmap 的扫描结果可以发现目前靶机开放了135、445、5985、8080以及49667端口

445端口

使用 smbclient 和 smbmap 查看默认共享


访问被拒绝，除非找到一些凭据否则将无法利用 smb 服务

8080端口


是一个 ManageEngine ServiceDesk Plus 站点，版本为 9.3

默认guest账户登录

在 ManageEngine ServiceDesk Plus 存在两个默认账户

在登录 administrator 无果后使用 guest 账户成功到登录应用中

在 Solution 选项卡中可以看到 Password Audit

点进去查看，发现存在一个 excel 表格

下载打开后如图所示，只有三个表

通过 binwalk 分解该 excel 表格，从中可以发现存在四张表，这表明原来有一张表隐藏了


选择工作表右键，点击取消隐藏发现Password Data工作表


其中包含一些密码和一个文件路径C:\temp\Password Audit\it_logins.txt，在路径对应的文件中可能包含一些登录信息

0x02 ME SDP 漏洞

这个版本的 ManageEngine ServiceDesk Plus 存在许多漏洞，我们可以通过这些漏洞来获取信息

XXE漏洞（CVE-2017-9362）

在 CMDB API 中的 INPUT_DATA 字段中添加 CI 时会造成 XML 外部实体注入攻击，攻击者在发送数据时将任何系统文件包含到 payload 当中，根据该漏洞编写了一个脚本

执行该脚本来抓取 win.ini


发现可以直接读取到 win.ini，那么也就能读取到上面的密码文件中的密码


成功拿到密码 alice/$sys4ops@megabank!

LFI/任意文件下载（CVE-2017-11511）

这次是通过本地文件包含漏洞来访问系统文件,但是目标文件作为与 SDP 安装关联的四个目录之一的相对路径需要给出,包括有意打开所有 SDP 安装的选项4，它允许请求所有与该路径相关的任何文件。与请求C盘的 win.ini 不同，SDP 安装在E盘。开始尝试获取数据库备份文件，首先请求日志路径为E:\ManageEngine\ServiceDesk\bin\SDPbackup.log，而访问路径如下


在其中记录了组成数据库备份的两个文件的位置，具体文件名如下

使用 wget 把它们下载下来进行查看


解压下载文件，里面存在许多 sql 文件


查看 aaapassword.sql 和 aaalogin.sql，其中包含各种账户的哈希值。

0x03 上线[system]

在 0xdf 大神的博客中讲述多条线路进行突破，分别是在 kali 下运用 administrator 账户获取 system 权限以及在 Windows 下使用之前获取的 Alice 用户通过 WinRM 进行权限提升，相比 Windows 的操作，我更喜欢使用 kali 来完成

rce完成上线

先前通过权限绕过获取到 administrator 用户，在 Admin 上存在 Custom Triggers 可以帮助我们完成远程命令执行

点击 Add New Action 来设置新的触发器

在 Action 中设置触发器脚本，将触发优先级设置为高并设置脚本命令帮助我们获取反弹shell


保存完成后在本地监听 443 端口，并开启 http 服务

在请求页面新建一个请求，将优先级设置为高
运行后返回 system 权限的命令行
