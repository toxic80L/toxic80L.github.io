HTB: Active

Active是一个简单的靶机，知识点涉及smb枚举、GPP解密、Kerberoasting、哈希解密等。smb匿名登陆后发现组凭证xml文件，经过解密后成功拿到用户SVC_TGS的密码，使用Kerberoasting可获取与普通用户相关联的服务用户名列表并返回票据信息，破解其中的哈希值拿到管理员密码，最终使用psexec上线系统权限。感兴趣的同学可以在HackTheBox中进行学习。

0x01 侦查

端口探测

首先通过nmap对目标进行端口扫描



扫描结果显示目标开放了88、135、139、445等端口，与此同时这还是一台域控主机

445端口

使用 enum4linux 对 smb 服务进行枚举



结果显示该目标是域控主机，同时存在默认共享

smb匿名登陆

使用 smbmap 再次对 smb 服务进行扫描


结果显示目前只有 Replication 能够匿名访问，使用 smbmap 递归列出其中所有文件


使用 smbclient 连接目标 smb 服务


经过一番搜索后在\active.htb\Policies\{31B2F340-016D-11D2-945F-00C04FB984F9}\MACHINE\Preferences\Groups\目录下发现重要文件Groups.xml

将其下载后在本地查看，其中包含域名、密码信息

这是一个GPP密码，在Windows系统中每当创建组策略首选项（GPP）时，都会在SYSVOL中创建一个包含该配置数据的xml文件，其中包括与GPP相关联的任何密码，为了安全起见，Microsoft AES将密码存储为cpassword，但随后微软在MSDN上发布密钥。与此同时，微软在2014年发布了一个补丁，阻止管理员将密码输入GPP，但是该补丁对已经存在的可破解密码无任何作用，渗透测试人员仍能发现之前保存的密码

由于密钥已知，我们可以直接使用 gpp-decrypt 进行解密


成功解密为GPPstillStandingStrong2k18

smb登陆用户共享

借助该密码我们还可以访问其他三个目录

使用 smbclient 访问用户目录

进入 SVC_TGS 的桌面查找第一个flag


成功下载并获取第一个flag

0x02 上线[system]

哈希获取

Kerberos是windows域环境中使用的身份验证协议（也可以对linux主机进行身份验证），早在2014年Tim Mdein提出了对Kerberos的攻击，整篇文章非常不错，通过图形说明了整个过程。当使用Kerberos对某些服务进行身份验证时，可以联系DC并告诉它需要向哪个系统服务进行身份验证，它使用服务用户的密码来哈希加密响应。而攻击者将该响应发送给服务，该服务可以使用它的密码来对其进行解密，从而检查身份并决定是否可以进入。可以通过离线破解与服务相关的密码，而不是将加密的票据从DC发送到服务，在大多数情况下需要一个活动账户才能启动Kerberoast，但如果DC配置来UserAccountControl设置“不要kerberos预身份验证”，则可以请求和接收票据以进行破解而无需域中的活动目录

重点关注以下几点

参考文章：https://www.qomplx.com/qomplx-knowledge-kerberoasting-attacks-explained/
