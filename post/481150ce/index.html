PHP代码审计

函数分析

由于目标站点模板采用的是 DocCMS，因此我们可以直接到目标官网当中下载源码，解压后利用 Seay 源代码审计工具进行漏洞审计

针对站点的主界面/index.php进行查看，其中可通过 lang 参数传递中文还是英文界面，指定/config/doc-config-为该 CMS 的配置文件。如果 CMS 尚未安装，则会包含/inc/nosetup/setup.html文件来进行自动安装，与此同时被包含的文件还有/admini/config/qd-config.php和/loader/load.php

/admini/config/qd-config.php是关于语言包的配置文件，重点来看看/loader/load.php，顾名思义其中都是加载页面的函数，但其中包含的/loader/doc.php引起了我的注意。

/loader/doc.php中发现cleanArrayForMysql函数针对请求值进行了过滤，$_REQUEST、$_GET、$_POST则代表客户端的请求方法，因此需要寻找cleanArrayForMysql函数的位置以确定该函数是否可绕过，同时也要寻找未被该函数过滤的请求值，因为其可能存在SQL注入漏洞

在/inc/function.php找到了cleanArrayForMysql函数，它开启了GPC转义，与此同时还发现了checkSqlStr函数，它已对存在SQL注入风险的敏感字符进行过滤。因此在之后的审计中也要关注checkSqlStr函数

SQL注入

在/inc/pay/alipay.php中发现其请求参数值并未受到cleanArrayForMysql和checkSqlStr这两个函数的保护，同时它被直接带入了SQL语句当中，因此存在SQL注入点

在/content/search/index.php中发现第二个注入点，虽然keyword参数被checkSqlStr函数所约束，但是通过urldecode函数可以帮助我们绕过限制

尝试使用search?keyword=1'经过一次URL编码，但是结果显示非法字符

尝试进行二次编码，结果目标显示报错语句，说明可能存在二次编码注入

使用 sqlmap 指定二次编码脚本进行测试，结果显示确实存在SQL注入漏洞


使用 sqlmap 查找库名

查看当前用户权限为普通用户，因此也无法直接获取权限。

查看用于管理员登录的代码文件/admini/login.php，其中的 checkPwd 函数首先会用户名进行判断，如果用户名存在才会检查密码，密码的检测需要依赖docEncryption函数

在/inc/class.docencryption.php中找到该加密函数，显而易见的是密码经过了 Sha1 以及 md5 双重加密，因此导致密码不可能逆向破解。

文件包含

在管理员主页admini/index.php中发现文件包含点，其中参数 l 通过GET方式传参，带入代码中就是doclang，它会与/../config/doc-config-目录拼接，但是由于.php的存在，我们需要通过00截断将该后缀去除。一旦找到上传文件，利用00截断配合文件包含可以完成代码执行，因此该漏洞比较鸡肋，需要满足PHP小于5.3和存在文件上传点这两个条件，否则无法利用

0x05 总结

整个靶场全面地展示了 Linux 内网渗透，通过对站点源码进行代码审计，发现 DocCMS 存在诸多漏洞，首先是SQL注入漏洞，但是拿到的密文无法解密，其次是后台建设脚本setup.php使用参数传递 MySQL 配置信息来导入数据，我们可搭建伪造的 MySQL 服务器用于后台脚本连接并成功读取网站配置文件，利用配置中的账号密码成功登录数据库，覆盖后台管理员密码哈希，因此可直接登录后台。后台存在模板编辑，通过该功能成功拿到用户权限。权限提升的道路也不简单，需要绕过PHP的diabled_functions才能执行命令，存在三条路径能够成功提权root用户，一是suid提权，二是 sudo提权，三是宝塔面板提权。拿到外网服务器的root权限后进入内网渗透环节，通过建立代理访问内网主机，访问其对应的 ddd5 域名，利用弱口令进入后台，发现模板处存在文件上传，使用菜刀设置代理连接木马，成功拿到内网主机权限，当然在内网主机中还发现了WDCP主机管理系统，进入该系统下载生成的SSH密钥，最终成功登录内网主机的root权限。该靶场非常贴近实战，十分适合网络安全进阶人员练习，在实战渗透时能够提供给我们非常好的借鉴作用。
