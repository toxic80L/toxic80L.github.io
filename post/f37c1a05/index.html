0x01 侦查

端口探测

首先通过nmap对目标进行端口扫描


但是扫描结果并没有返回端口信息，因此可能目标开放了udp端口，尝试使用 nmap 再次扫描

结果显示目标开放了69端口

69端口-udp

一般来说UDP的69端口为 tftp 服务，与 ftp 类似的是，它主要用于在远端服务器和本地主机之间传输文件，但tftp 相对于 ftp 来说比较简单，它没有 ftp 复杂的交互存取接口和身份认证机制，因此可以认为是 ftp 的简化版本。

由于不需要验证客户端的权限，我们可以使用命令直接登录 tftp

查看eula.txt发现目标操作系统版本为 Windows XP sp3

0x02 上线[system]

背景知识

mof

mof 文件格式可以使用文本格式来存储 WMI 数据，它可以由 mofcomp.exe 编译并存储在 windows WMI 存储库中。此外还有一个 mof 自安装目录%SYSTEMROOT%\System32\wbem\mof\，在该目录中任何mof文件都会被自动处理，在注册表中的默认值为HKLM\SOFTWARE\Microsoft\WBRM\CIMON\。

WMI

WMI是一组复杂的接口，主要用于在 Windows 操作系统上管理数据和操作，由存储在”WMI存储库”中的类和对象组成

震网（ms10-061）

该漏洞提供了在主机上作为打印机后台处理程序服务写入文件的能力，与此同时它将写入文件能力与允许代码执行的 mof 自安装目录相结合，并最终完成远程代码执行。以下是震网的相关源代码

通过分析 MSF 中的漏洞利用源代码，我们可以它发现经过一系列设置后连接至打印机，首先将 exe 形式的 payload 写入目标，之后将运行前者 exe 的文件的 mof 写入自安装目录中，通过 mof 的自安装特性运行第一步放置的程序，从而完成攻击。

mof利用代码

虽然该靶机中不涉及打印机和文件共享，自然也不涉及震网，但是我们需要了解震网中 WMI 所使用到的几个类，并构建出能够完成漏洞利用的 mof 代码

  • EventFilter：允许定义一些Windows事件，可以视为触发器
  • EventConsumer：消费者定义将要采取的行动
    • ActiveScriptEventConsumer：执行脚本
    • CommandLineEventConsumer：执行一个过程
    • LogFileEventConsumer：写入日志文件
    • NTEventLogEventConsumer：写入windows事件日志
    • SMTPEventConsumer：使用SMTP发送电子邮件
  • FilterToConsumerBinding：将触发器的实例和消费者的行动链接起来

0x04 总结

该靶机的利用方式非常新颖，由于tftp没有身份校验机制，我们可以直接登录并查看其中的文件。经过一轮信息收集后发现目标操作系统版本为Windows XP且对系统目录拥有写入权限，因此可以利用mof的自安装特性，将恶意mof至自安装目录，安装完成后会自动运行命令执行反弹shell，成功获取系统权限后无法读取flag信息，根据提示信息最终我们通过NTFS数据流成功拿到flag。
