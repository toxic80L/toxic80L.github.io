HTB: Jeeves

Jeeves是一个中等难度的靶机，知识点涉及Jeekins远程代码执行漏洞、KeePass解密、哈希传递、NTFS数据流读取等。Jenkins存在接口可用于远程代码执行，KeePass加密数据库解密能够获取管理员哈希值。感兴趣的同学可以在HackTheBox中进行学习。

0x01 侦查

端口探测

首先通过nmap对目标进行端口扫描


扫描结果显示开放端口为80、135、445以及50000端口

80端口


网页中存在搜索框，尝试进行搜索出现报错，报错信息显示其中存在SQL server 2005数据库

50000端口

在80端口未找到利用点，查看50000端口的 Jetty 中间件，版本为9.4.z

使用 searchsploit 寻找 Jettty 的相关漏洞


结果显示存在一个目录穿越漏洞，但是未能成功利用

目录扫描

尝试对80端口和50000端口进行目录扫描，寻找可能存在的应用


在50000端口中找到 askjeeves 目录，访问http://10.10.10.63:50000/askjeeves

结果发现这是 Jenkins 应用，版本为2.87

0x02 上线[kohsuke]

Jenkins是一款提供软件开发的持续集成服务的工具，它运行在Servlet容器中（例如Apache Tomcat）。它支持软件配置管理（SCM）工具（包括AccuRev SCM、CVS、Subversion、Git、Perforce、Clearcase和RTC），可以执行基于Apache Ant和Apache Maven的项目，以及任意的Shell脚本和Windows批处理命令

寻找Jenkins漏洞

使用 searchsploit 查找相关漏洞


在其中并没有找到版本为2.87相关的漏洞。但是经过浏览发现用户admin

远程代码执行漏洞

在尝试弱口令无果后发现不登陆情况下也能进行操作，于是在New Item中创建一个项目test

在Build Environmnet中选择Execute Windows batch command来执行 ping 命令


完成后点击build now并查看输出结果

成功执行命令，说明目标存在命令执行漏洞。开始构造用于反弹shell的powershell脚本

使用 nishang 中的 Invoke-PowerShellTcp.ps1 脚本进行反弹

在脚本最后添加地址和端口进行反弹，同时在当前目录开启 http 服务

在本地监听5555端口

在项目中设置执行命令用于反弹shell



完成后点击build now

成功拿到反弹shell，在当前用户桌面上寻找第一个flag


成功拿到第一个flag

检查系统配置

首先使用 PowerUp.ps1 对操作系统配置进行检查

KeePass密码破解

通过搜索在当前用户的文档目录下发现一个 KeePass 加密数据库


由于阅读该文件需要密码或密钥，我们需要寻找 .key 文件


结果并没找到该类型文件，于是选择使用smb协议进行下载，首先在 kali 中开启smb服务


在靶机中执行命令传输CEH.kdbx



使用 keepss2john 将 kdbx 数据库文件转换为可识别的哈希值


使用 john 进行密码破解


结果显示密码为 moonshine1，那么接下来需要通过该密码打开数据库文件

哈希传递

使用 keepass2 来打开 kdbx 文件


输入密码后可打开文件

按ctrl + h可以显示密码，拿到的NTLM哈希值如下

使用 smbmap 进行测试后成功访问，说明哈希值正确


使用 pth-winexe 来通过pth方式进行连接
