HTB: Bart

Bart是一个中等难度的靶机，知识点涉及密码爆破、网页信息爬取、javascript代码分析、登陆凭证获取、任意注册等。站点中存在密码爆破漏洞，爆破成功后可登陆系统，js代码审计分析后发现可通过日志接口上传木马，借助木马完成反弹shell，在靶机中发现注册表中存在管理员自动登陆凭证，借助该登陆凭证成功上线管理员。感兴趣的同学可以在HackTheBox中进行学习。

js代码分析

查看页面源代码，在其中发现一段 javascript 代码


通过测试发现当用户点击 logo 时，就会弹出done，然后弹出数字1。这是因为 xhr 是由三个参数进行调用，当该参数为 true 时将调用异步模式，脚本完成会提示”Done”，然后当http请求返回时，将调用 xhr.onreadystatechange 函数，该函数通过响应进行提示。与此同时它会将请求包直接记录到日志当中

日志记录获取webshell

访问http://internal-01.bart.htb/log/log.txt，其中记录了用户请求时间、参数以及user-agent等请求信息

那么如果将user-agent进行修改，同时修改 GET 请求中的参数 filename 是否能解析脚本呢？

首先使用 BurpSuite 截取点击log后的数据包，将参数 filename 修改为 phpinfo.php，user-agent设置为php脚本内容<?php phpinfo();?>

发送成功后访问http://internal-01.bart.htb/log/phpinfo.php

成功解析为php脚本语言，这就意味着我们可以直接获取webshell，将参数 filename 修改为 mac1.php，内容为一句话木马
