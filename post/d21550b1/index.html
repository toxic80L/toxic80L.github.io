HTB: Beep

Beep是一个简单的靶机，知识点涉及Elastix本地文件包含漏洞、sudo提权、Shellshock等。感兴趣的同学可以在HackTheBox中进行学习。

通关思维导图

0x01 侦查

端口探测

首先通过nmap对目标进行端口扫描


扫描结果显示目标开放了22、80、443、3306等端口

80/443端口

访问后发现这是一个 Elastix 登录页面，Elastix是一款专用交换机 (PBX) 软件。PBX 控制公司网络内的电话/语音网络，将其连接到网络的其余部分。

使用 gobuster 对当前页面进行目录扫描，成功扫描到多个目录

重点关注/admin目录，访问后出现登录提示

登录失败点击取消后重定向至https://10.10.10.7/admin/config.php，同时页面显示 FreePBX 的未授权信息

利用 searchsploit 搜索 Elastix 中的相关漏洞

结果显示目标可能存在本地文件包含漏洞，查看该漏洞的详细信息

漏洞利用代码中显示了目标POC信息vtigercrm/graph.php?current_language=../../../../../../../..//etc/amportal.conf%00&module=Accounts&action，该漏洞主要因为current_language参数未过滤，导致攻击者可以利用../目录穿越到配置文件，通过%00截断字符串忽略.php。直接利用该POC 访问目标，其中包含了一堆配置文件信息，获取到的账号密码如下：

使用拿到的账号密码登录 Elastix

10000端口

访问后为 webmin 登录界面，webmin 是用于管理 Unix 系统的 web 界面，尝试使用弱口令登录，但是结果显示失败。

25端口

通过本地文件包含，我们成功从/etc/passwd中读取到了用户列表

使用 smtp-user-enum 测试用户连接，结果显示目标用户都存在

0x02 上线[root]

该靶机存在多种方式提至 root 用户权限，下面将介绍其中的五种

Elastic远程代码执行

上线[esterisk]

除了本地文件包含漏洞，Elastix 还存在远程代码执行漏洞，但是直接利用会报错，因此需要略作改动

首先需要修改目标地址、本地地址以及本地监听端口

其次忽略 SSL 证书错误，将其传递至请求当中

最后则需要开放一个扩展，使用 svwar 进行寻找特定的 PBX 实例

将拿到的结果 233 附加到脚本的extension当中，通过 sslscan 对靶机支持的 SSL 协议进行扫描，结果显示靶机支持的 SSL 协议为TLSv1.0，这大大低于本机允许的范围

因此最佳的做法是在本机中修改 openssl 的配置文件，将执行的MinProtocol以及CipherString都修改为 None 值

运行利用脚本，成功收到反弹shell

权限提升[root]

漏洞利用脚本中建议使用sudo nmap完成 root 提权，因此我们可以借助 nmap 的![command]完成权限提升。


成功拿到对应的 flag

查看 sudo 的运行权限


其中存在多个命令不需要输入密码便可以 root 权限运行，包括chmod以及chown等修改文件权限的命令，我们可以使用 chmod 来修改/bin/bash的运行权限

现在只需要运行 bash 便可拿到 root 权限

webmin登录

通过本地文件包含成功拿到了目标的密码，通过root/jEhdIekWmdjE可成功登录 webadmin，该界面用于管理机器，并且具有完全 root 权限。

在其中以 root 权限创建任务执行反弹shell指令

过几分钟以后成功收到反弹shell

但目标回显未指定密钥交换方法，那么使用命令指定密钥交换方法再次连接

Shellshock

假设目前我并没有密码，那么我肯定会尝试登录 webmin，当我尝试登录时，目标会创建一个 POST 请求

任何时候只要存在 CGI，尤其是在旧机器上，我们就可以尝试使用 shellshock。将User-agent中的漏洞字符串修改为() { :; };[cmd]，首先测试睡眠命令，比如sleep 10，结果目标等待10秒后才返回，说明存在 shellshock 漏洞

在本地监听 icmp 数据包

在命令中执行 ping 指令，测试是否出网

成功收到 icmp 响应

尝试在命令中执行反弹shell命令

成功收到反弹shell

webshell上传

因为目标靶机存在本地文件包含漏洞，那么我只要在主机上放置一个 webshell，利用文件包含就可以实现权限获取，而上传文件则有多种方式

前两种方式经过测试后都无法上传文件，而发送电子邮件是一个不错的方法，其绝对路径路径为/var/mail/[username]，通过 swaks 发送电子邮件

利用文件包含漏洞读取文件，执行 id 命令成功回显

0x03 总结

Elastix 是靶机的主要突破点，通过 searchsploit 可以轻松找到该应用的历史漏洞，经过测试后发现目标存在本地文件包含以及远程代码执行漏洞。利用本地文件包含获取配置文件，成功拿到密码信息。通过远程代码执行漏洞拿到 esterisk 权限，在提权方面利用脚本建议通过 sudo 进入 nmap 的交互模式能够成功执行 root 权限命令，这时候提出思考：既然 sudo nmap 可以执行，那么其他命令是不是也可以通过 sudo 来无密码获取权限。结果当然是可行的，利用 chmod 修改 bash 权限即可轻松获取 root 权限。

其他获取管理员权限的方式还有很多，利用文件包含拿到的密码可以轻松登录 webmin 应用和 SSH 服务，且权限都是 root，在 webmin 中还因 CGI 接口发现了该应用存在 shellshock 漏洞，该漏洞通过 http 传输请求，因此也能直接获取 root 权限。最后一种方式则利用邮件服务，将带有木马的邮件发送至主机用户的邮箱中，通过文件包含执行命令，从而完成了 webshell 的基本操作。
