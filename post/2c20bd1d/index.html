HTB: Access

Access是一个简单的靶机，知识点涉及Access数据库、Telnet登陆、lnk文件分析、Runas提权、DPAPI信用提权等。感兴趣的同学可以在HackTheBox中进行学习。

0x01 侦查

端口探测

首先通过nmap对目标进行端口扫描


扫描结果显示目标开放了21、23以及80端口

80端口

访问后发现是一个包含 LON-MC6 的机房图片

使用 gobuster 对进行目录扫描，但是并没有发现什么可利用点。

21端口

由于目标存在 ftp 匿名登录漏洞，使用匿名用户anonymous/anonymous可直接登录ftp

进入 ftp 后存在两个目录，分别为/Backups和/Engineer，/Backups目录下存在数据库backup.mdb，而/Engineer目录下存在文件Access Control.zip

分别进入不同目录将数据库文件和压缩包下载至本地

DPAPI获取管理员密码

DPAPI（数据保护应用程序编程接口）是一种简单的加密应用程序编程接口，理论上，数据保护 API 可以对任何类型的数据进行对称加密；实际上，它在 Windows 操作系统中的主要用途是执行非对称私钥的对称加密

通过 DPAPI 信用获取管理员密码的条件则要借助系统中的用于加密用户 RSA 密钥的 DPAPI 密钥、用于保护用户私钥的主密钥以及用户凭据文件。默认情况下 DPAPI 密钥存储在%APPDATA%\Microsoft\Protect\{SID}目录下，其中 {SID} 是该用户的安全标识符，DPAPI 密钥与主密钥存储在同一文件中，我们可使用whoami /all获取当前用户 SID

进入对应的 SID 目录并发现 DPAPI 密钥0792c32e-48a5-4fe3-8b43-d93d64590580

使用 certutil 对其进行 base64 编码并导出为output

DPAPI 密钥经过 base64 编码后如下：

与此同时找到对应的用户凭据文件51AB168BE4BDB3A603DADE4F8CA81290

同样对用户凭据文件进行 base64 编码并导出

用户凭据文件经过 base64 编码后如下：

通过命令base64解码 DPAPI 密钥和用户凭据文件

使用 mimikatz 解密 DPAPI 密钥

mimikatz 会自动保存解密后的masterkey，利用它可解密用户凭据文件，当然也可以直接通过命令来指定masterkey，比如/masterkey:b360fa5d...

成功拿到明文密码55Acc3ssS3cur1ty@megacorp，使用该密码可成功以管理员身份登录 telnet

0x04 总结

Access 数据库是一种轻量级数据库，靶机以此命名说明其中需要利用该数据库获取数据。果不其然通过 FTP 匿名登录我们在其中发现了它，顺带收获一个 zip 压缩包。利用 mdbtools 可收集 Access 数据库的信息，成功在其中找到用户认证的账号密码，恰好解压 zip 压缩包需要该密码，压缩包中存在 outlook 邮件，但是格式显示为 pst，该格式无法直接查看，因此需要将其转换为 Mbox 格式。完成后借助 mutt 可查看邮件内容，邮件中包含系统用户的账号密码，利用账号密码成功登录 telnet 获取权限。

权限提升则存在两种方式，一是通过信息收集发现管理员凭据信息，二则是通过 dpapi 信用获取管理员密码。在 Public 用户桌面上存在 lnk 文件，使用 pylink 分析后发现管理员凭据已缓存，因此借助它可成功拿到管理员权限；相对来说第二种方法则比较困难，它的利用前提是需要获取加密用户 RSA 密钥的 DPAPI 密钥以及用于保护用户私钥的主密钥，通过 Mimikatz 解密后拿到管理员密码，登录 telnet 同样能够拿到管理员权限。
