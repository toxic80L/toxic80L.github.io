icacls读取flag

icacls 可以显示或修改指定文件上的自由访问控制列表（DACL），并将存储的 DACL 应用于指定目录中的文件。简单来说就是该应用嫩个够控制文件的权限。

在当前权限下使用 icacls 查看管理员桌面，发现当前用户实际拥有控制权限

虽然对root.txt没有读取权限，但是使用 icacls 可为当前用户添加读取权限


成功拿到第二个flag

0x03 权限提升[administrator]

注册表获取凭证

在注册表中发现用于自动登录的凭证密码：Welcome1!


考虑到管理员可能会将该密码重用，我们需要使用该凭证登陆管理员账户

ssh反向连接

首先将端口转发工具 plink.exe 上传至靶机

使用 powershell 命令在用户目录中下载 plink.exe

在 kali 中开启 ssh 服务

在靶机上通过 plink.exe 完成ssh端口转发


但是经过尝试后无法连接，我们可尝试使用登陆凭证提权

登陆凭证提权

由于当前命令行环境为 cmd，为了获取 powershell 环境，我们可使用 nishang 中的 Invoke-PowerShellTcp.ps1 脚本

在脚本最后添加本地地址和端口

同时在当前目录开启 http 服务

在本地监听1234端口

在靶机中执行命令并成功拿到反弹的 powershell

远程加载 PowerSploit 中的 PowerUp 脚本进行检测

使用administrator/Welcome1!作为登陆凭证

将 Invoke-PowerShellTcp.ps1 重命名为shell.ps1，和之前一样在最后写入反弹地址和端口，以便执行反弹shell

在本地开启7777端口进行监听

使用 powershell 下载shell.ps1并利用当前管理员凭证执行反弹shell

成功拿到管理员权限

成功获取第二个flag

总结：通过Achat系统存在远程代码执行漏洞我们成功拿到了用户权限，靶机对文件权限未配置得当，因此我们可以使用icacls来为当前用户添加读取权限，从而成功读取root.txt。powerup脚本能对靶机配置进行检查，发现在注册表中存在管理员自动登陆凭证，我们可在powershell环境下利用该凭证上线管理员并最终拿到靶机的管理员权限。
