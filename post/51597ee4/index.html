HTB: Heist

Heist是一个简单的靶机，知识点涉及Cisco密码破解、smb枚举、WinRM枚举、RPC密码访问、Firefox进程转储等。站点中存在信息泄漏漏洞，其中包含Cisco账号密码信息，将哈希值破解后枚举smb服务，找到账号密码登陆IPC并获取系统用户的SID，爆破SID从而获取用户列表并使用爆破成功后的字典枚举WinRM服务，枚举成功后登陆用户，在靶机存在Firefox进程，将其转储后能够直接读取管理员密码，最终通过WinRM登陆管理员。感兴趣的同学可以在HackTheBox中进行学习。

0x01 侦查

端口探测

首先使用 nmap 进行端口扫描


扫描结果显示目前目标开放了80、135、446、5985、49669端口

445端口

使用 smbclient 进行匿名登录


结果显示无法连接smb服务，匿名登录失败

80端口


访问后是一个登录界面，可以通过 guest 账户登录

Hazard 用户在求助 Cisco 路由器的相关问题，点击包含其中的附件

密码破解

在附件中我们发现其中存在账号和密码，分别是类型为 Cisco 7 和 Cisco 5 的哈希值

Cisco Type 5

使用 john 对 Type 5 哈希进行爆破


成功破解，密码为stealth1agent

Cisco Type 7

针对 Type 7 的哈希，可以借助在线网站进行破解
破解网站：https://www.ifm.net.nz/cookbooks/passwordcracker.html



经过解密后密码为rout3r/$uperP@ssword、admin/Q4)sJu\Y8qz*A3?d

Type 7 存在静态加密密钥tfd;kfoA,.iyewrkldJKD以及弱算法加密。我们可以直接使用 python 脚本进行破解

smb枚举

有了账号密码，那么我们就可以对先前的smb服务进行爆破，使用 crackmapexec 对其进行爆破


爆破成功，使用账号密码hazard/stealth1agent查询smb共享


当前用户只能读取$IPC，$IPC主要用于进程通信的共享，通常会出现空连接会话，但是目前我们需要凭证才能读取

使用 lookupnames 获取用户sid


也可以通过sid反查用户

由此可知我们可以通过暴力破解sid来获取用户列表

使用 impackets 中的 lookupsid.py 能够帮助我们更加方便地获取sid


成功获取到如下用户名

0x02 上线[chase]

登陆WinRM服务

目标开放 WinRM 服务，我们使用新的用户名和密码对其进行爆破


chase/Q4)sJu\Y8qz*A3?d能够成功登陆，使用 evil-winrm 完成登陆


在当前用户桌面下找到第一个flag


成功拿到第一个flag以及一个有趣的提示

0x03 权限提升[administrator]

Firefox获取凭证

我们注意到在该靶机上还存在 Firefox 浏览器，说明其中可能存在利用点


查看进程也证实了我们的想法：Firefox正在运行中，里面可能包含存储的明文密码

上传 procdump.exe 至C:\Windows\System32\spool\drivers\color目录下，也可以使用 Powersploit 中的 Out-Minidump.ps1 工具，只要能将火狐浏览器进程 dump 下来即可


使用 procdump 将 Firefox 其中一个进程从内存中 dump 下来


将其下载下来并在内存中寻找POST请求


下载成功，但是我在本地找不到相关文件。那么可以采取另外一个方式：上传 strings64.exe 将dmp文件转换为文本格式

在生成的文本中查找密码


密码为4dD!5}x/re8]FBuZ

登陆WinRM服务

通过获取到的账号密码登录 WinRM 服务

在管理员桌面上寻找第二个flag


成功获得第二个flag
